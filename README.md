# Linky

> 把散落在屏幕里的工作，整理成随时可以继续的记忆。

Linky 是一款面向 Windows 的本地记忆与工作接续工具。它在获得你的明确授权后，安静地理解你正在做什么，把值得保留的工作片段自动整理成清晰的事件。下次被消息、会议或新任务打断时，你不必重新翻遍窗口和聊天记录，就能快速找回：**我刚才在做什么、做到哪里、下一步是什么。**

**整个核心链路都在你的电脑上运行。** 采集、文字提取、事件整理、搜索索引与可选模型推理均在端侧完成；Linky 不要求登录，不把记忆上传到云端，也不使用遥测收集你的工作内容。

它还有一只常驻桌面的伙伴。它不负责聊天，也不会抢走注意力，只用轻微的动作和适时提醒告诉你：Linky 正在工作，重要的事情没有丢。

## Linky 能为你做什么

### 自动整理，而不是让你整理

Linky 从窗口状态、系统可访问文本和必要时的局部画面中提取线索，再把零散信息聚合成完整事件。你看到的是整理后的目标、进展、事实和下一步，而不是一长串原始截图或杂乱记录。

### 一键回到上次的位置

事件拥有可靠来源时，点击 **继续工作**，Linky 会帮助你重新打开相关页面、文件或应用；无法安全定位时，它会明确说明，而不是猜测或打开错误内容。

### 把上下文交给任何 AI

点击 **复制记忆**，即可获得一份结构化的记忆 Prompt，其中包含任务目标、当前进展、关键事实、约束和建议下一步。粘贴到你常用的 AI 工具里，就能少解释几遍，直接接着做。

### 搜索你做过的事

用自然语言或关键词查找曾经处理过的项目、页面与问题。Linky 会从已经整理好的事件中找答案，而不是要求你记住文件名或准确日期。

### 有分寸的桌面陪伴

桌宠会根据记录状态和工作阶段呈现不同反应，在适合接续任务时进行低频提醒。它可以拖动、可以安静，也不会读取原始隐私内容或变成另一个打扰你的聊天窗口。

## 你的记忆，只属于你

Linky 从设计之初就把隐私边界放在功能之前：

- 核心能力完全端侧运行，无需把工作内容发送给远程服务。
- 无需账号，不做云同步，不发送内容遥测。
- 未经明确同意不开始记录，暂停后不产生新记忆。
- 密码输入、锁屏、私密浏览和用户排除项在采集源头跳过。
- 截图、提取文本、事件与索引保存在本机，并由用户随时删除。
- 可选模型也在本机推理；模型不可用时，基础记忆能力仍然有效。

没有任何软件能够负责任地承诺绝对“零风险”，但 Linky 尽量让隐私风险不离开你的设备，并把记录、排除、保留和删除的决定交还给你。

## 为什么是 Linky

- **结果优先**：自动生成事件，用户不需要每天维护一套复杂的知识库。
- **端侧运行**：记忆、截图、索引与可选模型推理都留在当前设备，不上传工作内容。
- **隐私可控**：未明确同意前不采集；支持随时暂停、排除应用与网站、删除事件和清空数据。
- **证据可追溯**：事实、模型推断和来源分开呈现，模型不会被当作绝对正确。
- **离线可用**：即使可选模型不可用，基础记录、时间线、关键词搜索和记忆复制仍可工作。
- **低打扰**：密码输入、私密浏览、锁屏和排除项默认跳过；桌宠在全屏、会议等场景主动安静。

## 一次典型使用

```text
你在浏览器、文档和聊天工具之间推进一个项目
                 ↓
Linky 在本机识别有价值的工作片段
                 ↓
零散片段被自动整理成一件完整事件
                 ↓
你随时查看、搜索、删除或继续这件事
                 ↓
点击“继续工作”，或复制记忆交给 AI 接着完成
```

## 目前的产品边界

Linky 目前专注于一件事：**可靠地记住工作，并帮助你继续。**

它不是自由聊天机器人，不会自动替你操作电脑或执行命令，也暂不提供云同步、多人协作和完整知识库。模型只负责增强整理结果；基础记录、查找、删除和复制能力不会依赖模型才能运行。

当前版本仍处于早期开发阶段。公开发布前还需要完成正式代码签名，以及更多 Windows 10/11 设备、不同显示缩放、升级安装和长时间运行验证。

## 开始开发

环境要求：Windows 10/11 x64、Node.js 22+、pnpm 11。打包或原生依赖无法命中预编译版本时，还需要 Visual Studio 2022 C++ Build Tools。

```powershell
pnpm install
pnpm check
pnpm --filter @memory-companion/desktop exec electron-builder install-app-deps
pnpm dev
```

构建 Windows 安装包：

```powershell
pnpm package:win
```

> 请先运行测试，再安装 Electron 原生依赖。测试环境与 Electron 使用不同的原生 ABI。

## 项目结构

- `apps/desktop`：Electron 主进程、预加载层、React 主界面与桌宠窗口。
- `packages`：数据契约、记忆生产、事件聚合、搜索、模型适配与桌宠状态机。
- `tests`：跨模块测试、真实场景夹具与端到端回归。
- `docs`：产品冻结基线、隐私边界、验收标准与实施记录。
